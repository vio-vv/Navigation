# 抽象地图生成路线

## 目标

逐步生成一张可用于路径规划和层级管理的抽象地图。当前阶段先验证点集分布，后续再加入边、核心点和行政层级。

## 阶段一：点集生成

- 使用泊松磁盘采样生成基础点集。
- 保证点之间有最小距离，避免局部过密。
- 支持通过参数控制地图边界、目标点数、最小距离和随机种子。
- 使用 `poisson_points.cpp -> poisson_preview.py` 的管道方式可视化验证。

需要关注：

- 目标点数是否能稳定生成。
- 点是否过于均匀，缺少真实地图中的密度差异。
- 边界附近是否出现明显空洞或突兀分布。
- (经验证，仅采用泊松磁盘采样，而不加其他处理，点集生成过于均匀，且边界出现明显空洞，泊松磁盘采样仅是生成基础点集，还需后续处理)

## 阶段二：核心点生成

- 在普通点集中选择或单独生成核心点。
- 核心点用于表示更高行政层级，例如省级、市级、区级中心。
- 核心点之间应保持更大的最小距离。
- 核心点不宜过于靠近边界。

可选策略：

- 先生成核心点，再围绕核心点生成普通点。
- 使用不同半径生成不同层级的核心点。
- 根据核心点影响范围给普通点分配行政归属。

## 阶段三：区域与层级划分

- 根据核心点将普通点划分到不同区域。
- 初期可使用最近核心点归属。
- 后续可加入权重，例如核心点等级、距离衰减、区域容量。
- 每个点维护 `address` 或类似层级编号。

需要验证：

- 每个区域点数是否合理。
- 区域边界是否过碎。
- 不同行政层之间是否能形成清晰包含关系。

## 阶段四：边生成

- 在点集基础上生成候选边。
- 初期可使用 k 近邻或 Delaunay 三角剖分。
- 再通过最小生成树保证全图连通。
- 删除过长、交叉过多或不符合层级规则的边。

边的类型可以逐步加入：

- 普通局部边。
- 区域内部主干边。
- 核心点之间的高层级连接边。
- 跨区域连接边。

## 阶段五：道路属性

- 为边添加长度、容量、流量和时间函数参数。
- 普通边容量较小，核心边容量较大。
- 根据行政层级或边类型设置不同权重。
- 支持后续路径规划和拥堵模拟。

## 阶段六：验证工具

- Python 可视化继续作为主要验证工具。
- 点、边、核心点和区域归属都通过标准输入或 CSV 输入绘制。
- 后续可以增加不同颜色表示行政区，不同线宽表示边等级。

建议验证项：

- 点距分布。
- 区域点数统计。
- 图是否连通。
- 平均度数和最大度数。
- 边长分布。
- 核心点之间是否连通。

## 当前优先级

1. 稳定 C++ 点集生成与 Python 可视化管道。
2. 加入核心点生成策略。
3. 加入基础边生成与连通性检查。
4. 扩展可视化，展示核心点、边和区域层级。
